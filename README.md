Collaborative Terminal Rich Text Editor in C (Client-Server)
This project is a real-time collaborative rich text editor that runs in the terminal. It uses a client-server architecture where a central server manages the document state and synchronizes changes among multiple connected clients.

File Structure
The project is organized as follows:

/ (Root Directory)

README.md: This file.

Makefile: Compiles the server and client executables.

server.c: The source code for the server application.

client.c: The source code for the client application.

/libs

common.h: Shared data structures (Document, Paragraph, etc.).

common.c: Shared implementation for JSON conversion functions.

mongoose.h: Header for the Mongoose networking library.

mongoose.c: Source for the Mongoose networking library.

/server-data (Generated by the server on first run)

coop.json: The live, collaborative version of the document.

pub.json: The "published" version of the document.

users.txt: A file containing hashed user credentials.

/versions: Directory for storing old versions of pub.json.

/temp: Directory for storing the last 30 versions of coop.json.

Prerequisites
You will need the following libraries installed on your system to compile the project:

libjansson-dev: For handling JSON data.

libncurses5-dev: For the terminal-based user interface.

libssl-dev: For SHA256 hashing (authentication).

On a Debian/Ubuntu-based system, you can install them with:

sudo apt-get install libjansson-dev libncurses5-dev libssl-dev

Compilation
Navigate to the root directory of the project and run the make command:

make

This will create two executables in the root directory: server and client.

Running the Application
1. Start the Server
First, run the server in one terminal. It will automatically create the server-data directory and a default users.txt file if they don't exist.

./server

The server has two hardcoded credentials initially:

A default admin user (admin / admin123) is created if users.txt is empty.

A server password (vodka) is required for new users to register.

2. Connect as a Client
In a separate terminal, run the client, providing the server's IP address, your username, and your password.

# Usage: ./client <server_ip> <username> <password>
./client 127.0.0.1 admin admin123

If you are a new user, the client will prompt you to enter the server password. Once authenticated, the terminal editor will launch.

Editor Controls
Arrow Keys: Move the cursor.

Enter: Create a new line.

Backspace/Delete: Delete characters or merge lines.

Tab: Insert 4 spaces.

Shift + Left/Right Arrow: Start selecting text.

ESC: Exit Select or Command mode and return to Edit mode.

Alt + c: Enter Command mode.

In Command Mode (:):

bold, ital, unline, strike: Apply/remove styling to the current selection.

quit: Exit the editor.
